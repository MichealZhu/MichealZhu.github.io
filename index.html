苹果铃声的更换

今天想换个铃声，一时想不起如何换了。百度一番后，想来还是该记录一下。
需要：itunes
1.下载好所需要的铃声（可以自己制作一个音频）。
2.连接手机打开itunes将音频文件拖入itunes歌曲列表（点击文件 加入资料亦可）。
3.在itunes中右键音频文件，显示简介,点击选项，选择所需要的音频片段，苹果规定
需要小于40s.（音频命名尽量不要出现字符）
4.进行格式的转换 itunes文件选项 点击转换 创建ACC版本 获取文件位置 将格式m4a更改
成m4r.
5.再在itunes上选取铃声一栏将音频文件拖入或添加至资料库
6.将文件拖入手机处即可 如图：（同步亦可）

闭包的简单理解

闭包

就对闭包的理解来说始于和函数内部的值外部无法直接获取 （单纯的获取方式：函数不调用不执行 1 调用 2 调用产生值 return将其送出）而函数内部可以获取 上下作用域 所以return 可以获取值
那么需要将值进行修改呢 可以用函数 所以return （出函数 函数里操作将值进行修改 将值return）
所以闭包的基本形式：一个外部函数和一个内部函数（return一个内部函数） 通过内部函数来访问更改外部函数的值
function test(){
var a= 1;
return function(){
return a=a+1;
}
}
闭包的作用：
1.给以一个安全的环境封闭的空间 能够让闭包中的值不会轻易改变（在全局中随意可以更改）和污染全局 需要通过给以的特定的方法获取或改变 （作用域的关系）

eg function test(){
var name=’zjl’;
}
test（）；此刻值已存在 只不过外部无法获取

如进行闭包的使用 则可以进行值的获取和更改 （想要更改我的数据必须通过我的方法 也可在此之中进行逻辑的校验）

2让数据永久存于内存之中

在这段代码中，result实际上就是闭包f2函数。它一共运行了两次，第一次的值是999，第二次的值是1000。这证明了，函数f1中的局部变量n一直保存在内存中，并没有在f1调用后被自动清除。

为什么会这样呢？原因就在于f1是f2的父函数，而f2被赋给了一个全局变量，这导致f2始终在内存中，而f2的存在依赖于f1，因此f1也始终在内存中，不会在调用结束后，被垃圾回收机制（garbage collection）回收。
